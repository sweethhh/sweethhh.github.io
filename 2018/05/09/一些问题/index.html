如何加快页面首屏显示速度？

  1. 简洁的代码，减少代码冗余，能用css实现的效果尽量不要用js。
  2. 减少资源的载入，避免为了实现一个效果去引用较大的资源库。
  3. 图片使用懒加载，懒加载是为了防止页面一被打开就立即载入图片，这样就会占用网络带宽，从而影响网站的打开速度以及网站首屏的请求，所以非正文图片都是可以做懒加载的。禁止视频或音频自动播放。
  4. 将css引用放在头部，js放在尾部，js采用async方式加载。对非文字元素指定宽高，避免浏览器重排重绘。
  5. 服务器端提供html文档和http静态资源时，尽量开启gzip压缩。
  6. 尽量减少HTTP Requests的数量。
  7. 图片无损压缩的优化。
  8. 减少DOM Elements的数量。
  9. CDN加速，CDN服务商在全国各地部署一些节点服务器，如果你的网站要使用他们CDN的话，他们就会把你网站里能够缓存的内容都缓存到各地的节点服务器上。这样不同地区、不同运营商的访问者访问你网站的时候，就可以就近在一个节点上访问，不需要去你源站。从而起到加速的效果。

.png和.jpg的区别？

.png支持透明，无损保存，多次保存都不会影响图片质量。但png较大。
.jpg不支持透明、动画，JPEG的压缩方式通常是破坏性资料压缩（lossy compression）,意即在压缩过程中图像的品质会遭受到可见的破坏。一张图片多次上传下载后，图片逐渐会失真。jpg一般较小。

什么是事件委托？

主要用于给大量重复的标签添加事件。
事件委托大致思想是给父元素添加一个监听事件，利用冒泡机制，再点击元素时判断是否是我们需要的元素。

可以通过一系列方法获知被点击的元素。
e.target是被点击的元素。

在jquery中，可以通过on()方法，给被选元素及子元素上添加一个或多个事件处理程序。
$(selector).on(event,childSelector,data,function)

px和rem和em区别?

px是相对于显示器屏幕分辨率而言的。
em是相对长度单位。相对于当前对象内文本的字体尺寸。子元素字体大小的em是相对于父元素字体大小。如当前对行内文本的字体尺寸未被人为设置，则相对于浏览器的默认字体尺寸。任意浏览器的默认字体高都是16px。
rem是CSS3新增的一个相对单位（root em，根em）,通过它既可以做到只修改根元素就成比例地调整所有字体大小，又可以避免字体大小逐层复合的连锁反应。需要给html标签设置一个font-size。

position的值类型

static:默认属性，元素正常出现在流中
fixed:生成绝对定位的元素，相对于浏览器的窗口进行定位
absolute:绝对定位，设置为该属性的元素会从文档流删除，相对于父元素定位
relative:相对定位，不删除文档流，相对于元素本来的位置进行定位
sticky:粘性定位，在没有滚动至超出范围的时候，就像relative，当滚出范围，就像fixed一样。
inherit:规定应该从父元素继承 position 属性的值。

display的值和类型

inline：默认
none: 隐藏且不保留原有占位
block：块级元素
inline-block：行内块级元素
flex:弹性布局，Webkit 内核的浏览器，必须加上-webkit前缀。采用此布局的容器，所有子元素自动成为容器成员。

浮动及清除浮动

浮动float会使元素脱离文档流，从而使父元素没有被撑起高度。

清除浮动：

  1. 新建一个空的DIV,属性设置为style="clear:both"
  2. 在父元素上设置overflow:hidden
  3. 给父元素设置浮动
  4. 父元素设置display：inline-block
  5. :after伪元素清除浮动，父元素设置:after,并定义display：block;visibility:hidden;clear:both;

get与post区别？

get:向指定资源请求数据
post:向指定资源发送需要被处理的数据

  1. post会在点后退时重复提交，get不受影响
  2. get可被收藏为书签，post不可以
  3. get会被缓存，post不会
  4. 在发送数据的时候，get的长度限制在2kb,post无限制
  5. get限制数据只能为ASCII码，post不限制
  6. post比get更安全，因为get会被缓存，post不会

WEB storage 和 cookie?

都是保存数据的一种方式。

  1. web Storage 是新的一种方式 ，对IE兼容不友好。
  2. web Storage不会发送数据到服务端，cookie会发送。
  3. cookie保存数据大小限制4k,webstorage限制5M
  4. cookie在设置过期前有效，localStorage在手动删除前都有效，SessionStorage在关闭浏览器前都有效。
  5. cookie和localStorage在同源窗口中共享，SessionStrage在不同浏览器窗口中共存。
